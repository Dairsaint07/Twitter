       There are multiple systems that we are going to be looking at. Some of them include MySQL, FlockDB, HDFS(Hadoop Distributed File System), and Mesos. These systems are a few that store twitters information. “Twitter uses MySQL heavily for primary storage of tweets and users”. FlockDB is a graph database but its built on MySQL. The main database used today that we will be looking at is called Manhattan. This was needed because they needed a database that could do a lot of queries in a short amount of time. The system is a database that is always running. This system has low latency and is very scalable. It is a predictable system, meaning that worst case performance is imperative. Overall they tried to make the system as simple as possible because it cannot have things go wrong with it since it must remain running 24/7. Manhattan has four layers to it, the interface,storage services, storage engine, and the core. This system seems very complex but it is actually as simple as they can get it with the goals that it has to achieve (scalability, latency, uptime, etc...).
